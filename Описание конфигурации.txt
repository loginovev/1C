Версии конфигурации: 1.0 для 1С 8.0, 1.1 для 1С 8.1, 2.0 для 1С 8.2, 3.0 для 8.3, 3.1 для 8.3 (управляемые формы).
Конфигурация внедрялась: ООО «Сиб-телеком», ООО «РадиоИмпульс-Телеком», ООО «Арт-Телеком»/ООО «Орион-Телеком».
Конфигурация «Оператор связи 3.1» предназначена для автоматизации биллинга операторов стационарной связи. Система позволяет тарифицировать звонки абонентов различными способами, рассчитывать условную стоимость для разных поставщиков международной и междугородной связи. Поддерживаются АТС М-200 и Квант. Возможно начисление фиксированной платы за пользование телефонной линией, интернетом, телевидением, домофоном.
Система поддерживает работу с фискальным оборудованием (ФР). Поддерживаются загрузки из различных систем приёма платежей. Механизм загрузки позволяет настроить получение данных из любого текстового файла. В реальной работе настройка производилась для систем: Платёжка, Сбербанк, Город, Qiwi. 
Конфигурация поддерживает обмен данными с поставщиками связи в форматах: Ростелеком, Транстелеком, Искра, Совинтел.
Система позволяет рассчитывать трафик между операторами связи. Например, между Демо-Телеком и Оператором междугородной связи (сотовые РФ и Москва) (смотрите демо-базу).
Поддерживается возможность работы с линейными данными. При этом указывается месторасположение аппаратуры и подключение к ней абонента.
Особенностью конфигурации является полный отход от рекомендаций 1С для хранения данных. Расчеты с абонентами хранятся не в регистрах накопления, а в справочниках для обеспечения высокой скорости работы с большими объёмами данных. При этом используется 13 справочников для хранения взаиморасчетов с абонентами, по одному на каждый месяц года и один справочник для архива. Аналогичная схема используется для хранения телефонных разговоров. Этим обеспечивается высокая скорость (порядка 0,2 – 0,3 секунды) при выборке акта сверки и оплаты для объёма базы 58Гб.
